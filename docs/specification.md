# いまいくら？ 仕様書

更新日: 2026-07-12

## 1. 概要

書店で選んだ本の合計金額を確認するスマートフォン専用Webアプリ。iPhone SafariとAndroid Chromeの最新版を対象とし、日本語、日本円、消費税10%に固定する。アカウント、独自サーバー、アクセス解析は使用しない。

## 2. 技術と公開

- Vite + React + TypeScript
- GitHub Pagesで公開する
- 正規URLは `https://imaikura.takagi.dev/` とする
- `imaikura` を独立した公開Gitリポジトリとして管理する
- MITライセンスを適用する
- 初回リリースではPWA、履歴、共有、予算設定、多言語、ダークモードを対象外とする

## 3. スキャンと入力

- 背面カメラを全画面で起動し、連続スキャンに対応する
- `978` または `979` で始まる有効なISBN-13だけを採用する
- 下段の書籍JANコードなどISBN以外は無視し、同一ISBNの重複追加を防ぐ
- 成功は画面で通知し、対応端末のみ短く振動させる。音は鳴らさない
- 対応端末のみライトボタンを表示する
- iPhone Safariでは振動APIが利用できないため、成功表示を必須の通知手段とする
- ISBN-10、ISBN-13、ハイフン付きISBNの手入力に対応し、内部ではISBN-13へ統一する

## 4. 書籍情報

- OpenBDだけを書誌情報源として使用する
- 表紙、タイトル、著者、税抜価格を取得する
- ISBN認識時点で仮項目を追加し、書誌情報は非同期取得する
- 未収録時はタイトルを任意、税抜価格を必須として手入力できる
- 通信失敗時はオンライン復帰時の自動再取得と手動再試行に対応する
- 手入力または修正済みの価格を自動取得で上書きしない

## 5. 一覧と合計

- 「購入予定」と「保留」の2セクションに分け、新しい本を先頭に表示する
- 「保留にする」「購入予定に戻す」ボタンでセクションを移動し、購入予定だけを合計に含める
- 個別削除後は「元に戻す」を表示し、全件削除は確認後に実行する
- 価格修正、変更済み表示、OpenBD取得価格への復元に対応する
- 価格未入力の本は合計から除き、未入力件数を表示する
- 税抜小計の10%を一度だけ計算し、1円未満を切り捨てる
- 税抜小計、消費税、税込合計を画面下部に固定表示する

## 6. 保存とプライバシー

- 一覧、保留状態、入力内容、価格修正を端末内へ自動保存する
- 選書情報を独自サーバーへ送信しない
- 「このアプリについて」にOpenBD、カメラ、端末内保存を記載する

## 7. デザイン

- 白と墨色を基調とし、主要操作を緑で強調する
- 本とバーコードを組み合わせたアイコンを使用する
- SVGファビコン、Apple Touch Icon、OGP画像を同じ意匠で提供する
- 空の状態では大きなスキャンボタンを表示する
- 片手操作、文字拡大、十分なコントラストに配慮する

## 8. 受け入れ条件

1. 対象ブラウザで背面カメラを起動・終了でき、終了後はカメラを停止する
2. 有効なISBNだけを連続で追加でき、重複や下段コードを追加しない
3. 手入力ISBNを検証・正規化できる
4. OpenBD取得中も次の本を追加でき、失敗時に再試行できる
5. 未収録・価格なしの本を後から補完できる
6. 購入予定、保留、削除、取り消し、全件削除が仕様どおり動く
7. 税抜小計、消費税、税込合計が仕様どおり更新される
8. 状態が端末内に保存され、再読み込み後に復元される
9. カメラ拒否、通信失敗、無効ISBN、情報未取得を区別して表示する
10. GitHub Pagesで直接アクセスと再読み込みが動作する
11. 自動テスト、型チェック、プロダクションビルドが成功する
12. iPhone SafariとAndroid Chromeで実物の本を使って実機確認する

## 9. 仕様更新方針

実装中に仕様を具体化または変更した場合、この文書を同じ変更単位で更新する。

## 10. 実装メモ

- OpenBD取得価格と利用者の入力価格を別に保存し、再取得による上書きを防ぐ
- ISBNを認識した時点で重複を同期的に判定し、短時間の連続認識も抑止する
- GitHub Actionsはテストとデプロイを別ワークフローにし、mainのテスト成功後だけPagesへ公開する
- 自動テストではISBN、税計算、OpenBD変換、端末内保存、書籍状態管理、主要なカード操作を検証する
- カメラ映像の認識精度、ライト、端末振動は実機受け入れ確認で検証する
- 実機確認結果は `docs/device-test-checklist.md` に記録する
- OGP、canonical URL、Twitter Card、robots.txtを公開用メタデータとして提供する
- GitHub Pages の独自ドメイン設定として `imaikura.takagi.dev` を `CNAME` で管理する
- `App`は画面構成、各コンポーネントは表示責務、Hookは状態と副作用に責務を分ける
- PrettierでTypeScript、CSS、Markdown、YAMLの書式を統一し、CIで検証する
